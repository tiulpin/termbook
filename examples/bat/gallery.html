   3 │ Generate browsable HTML galleries from CLI terminal output. Like Storybook, but for command-line tools.

   5 │ Captures ANSI-colored output and renders it as a static page with a grid layout, dark/light themes, click-to-expa

  10 │ ## Examples
  11 │
  12 │ - [eza — modern ls](https://htmlpreview.github.io/?https://raw.githubusercontent.com/tiulpin/termbook/main/exampl
     │ es/eza/gallery.html)
  13 │ - [ripgrep — fast recursive search](https://htmlpreview.github.io/?https://raw.githubusercontent.com/tiulpin/term
     │ book/main/examples/ripgrep/gallery.html)
  14 │
  15 │ Manifests live under [`examples/`](examples/); each one is a `.termbook/termbook.yml` plus committed `*.ansi` cap
     │ tures.
  16 │
  17 │ ## Usage
  18 │
  19 │ ```go
  20 │ book := termbook.New("mycli — screen gallery",
  21 │     termbook.WithGitHub("https://github.com/you/mycli"),
  22 │ )
  23 │
  24 │ book.Category("Users", "users",
  25 │     termbook.Scr("user-list", "user list", "List all users",
  26 │         "mycli user list", capturedANSIOutput),
  27 │ )
  28 │
  29 │ book.Generate("docs/gallery/index.html")
  30 │ ```
  31 │
  32 │ `Scr` takes an ANSI string — capture it however you want. There's also `Manual` for building output with a writer
     │ :
  33 │
  34 │ ```go
  35 │ termbook.Manual("login", "login", "Auth flow", "mycli login", func(w io.Writer) {
  36 │     fmt.Fprintf(w, "✓ Logged in as admin\n")
  37 │ })
  38 │ ```
  39 │
  40 │ ## Capture helper for Cobra
  41 │
  42 │ ```go
  43 │ func capture(rootCmd *cobra.Command, args ...string) string {
  44 │     var buf bytes.Buffer
  45 │     rootCmd.SetArgs(args)
  46 │     rootCmd.SetOut(&buf)
  47 │     rootCmd.SetErr(&buf)
  48 │     rootCmd.Execute()
  49 │     return buf.String()
  50 │ }
  51 │ ```
